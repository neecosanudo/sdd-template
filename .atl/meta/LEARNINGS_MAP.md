# Learnings Map

> **Purpose**: Track insights discovered in other projects that can strengthen this template. Every project teaches something — this map ensures those lessons survive and propagate.
>
> **Scope**: This file is part of the template's meta-infrastructure. It applies to ALL projects using this template, but it tracks learnings that improve the TEMPLATE itself.

## How to Update This File

### Manual Entry
When you discover a pattern, tool, or convention in another project that would improve this template:

1. Assign the next sequential learning ID (`LEARN-NNNN`)
2. Fill all required fields
3. Set status to `identified`
4. Evaluate whether it should be integrated

### Automated Scanning
You can ask your agent to scan directories on your machine:

```
"Scan ~/projects for .md files that map where information is stored 
 (like READMEs, CONTRIBUTING, or index files). Update LEARNINGS_MAP.md 
 with any new patterns that could strengthen the template."
```

The agent SHOULD:
1. Search for `.md` files in the specified directories
2. Look for files that serve as "information maps" or "project indexes"
3. Compare findings against existing template structure
4. Propose new entries for patterns not yet captured

### Review Cadence
Review this file:
- After completing any project ("What did we learn?")
- When onboarding a new project type ("What patterns apply?")
- Quarterly ("Are there stale entries to archive?")

## Learning Lifecycle

| Status | Definition | Who moves it |
|--------|-----------|--------------|
| `identified` | Learning observed but not yet evaluated | Anyone |
| `evaluated` | Assessed for template fit and impact | Template maintainer |
| `integrated` | Incorporated into template (file changed) | Template maintainer |
| `verified` | Confirmed working in at least one real project | Template maintainer |
| `rejected` | Decided NOT to integrate (with reason) | Template maintainer |
| `archived` | No longer relevant, kept for history | Template maintainer |
